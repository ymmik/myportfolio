Optimizing UX and Revenue:
An Enhanced Strategy for an SEO Tool

UI/UX Design • User Research • Strategy Proposal

OVERVIEW
Begin with assumption-driven design, followed by further research based on the design outcomes for an SEO Tool.
ROLE
Responsible for UX/UI design, user research, and strategy proposal.As the only designer on the team, I collaborated with the product manager and engineers on the project.

GOAL & ASSUMPTION

Goal   Increase weekly active users, usage frequency, and retention rate.
Assumptionn   Adding features like keyword search trends and related keywords will enhance performance.

UI/UX Design

We added the new features: providing keyword search trends and related keywords, to our keyword research tool. Also, we changed the visual style and rearrange the interface to make it more clear and more understandable.

Design Outcome

Number of the weekly active users

The number of paid users increased by 24.6%, and free trial users increased by 19.8% compared to the previous month.

Weekly usage

The amount of usage grows 24.1% for the paid user, and 70.4% for free trial users compare to the previous month.

Retention rate

The retention rate doesn't really improve.

( Average number of weekly active users, from web analysis tool Mixpanel )

In conclusion, the number of users and the amount of usage are improved, yet the retention rate is not. This implies the new function we provided might not be able to motivate users to keep on using our product, they probably don't need the feature as frequently as we expected. However, the new function did help improve the overall usage of the product obviously. If we want to get a higher retention rate, we should try some other ways.

Next Step: User Research

According to the outcome of the keyword research tool revision, we found even though users did increase after adding new features, it did not reflect an improvement in retention. We can suggest—new features might not be critical features that attract users to keep on using, it’s probably just a nice-to-have feature. Still, it’s not easy to confirm if the conclusion is correct, since the team hasn’t really done any user research before. We need to help the team know real users’ needs, and make a better decision on product direction. To do better decisions on product revision, we need to know more about our users. It's necessary to conduct user research and build a persona to figure out real users' needs, which might help to find out ways to improve user retention.

To find out what users really need, we need more information. Besides observing the existing user data we have, we also need to conduct surveys and interviews to define our user persona.

1. Find insights from user behavioral data

Some insights we’ve found from data:

(1) Most paying users unsubscribe within only a few months. However, it takes a longer time to see the result of SEO in fact.

(2) The amount of usage doesn’t seem to indicate user lifetime. Many long-term users are even with less usage and lower frequency than short-term users.

(3) Most users have tried all main functions. Among all used functions, most users use keyword research and webpage SEO check function more than others.

2. Survey Design

By doing a survey, we can know what functions users care about, how they use them, and collect information for the user interview question design.

Target: Paying user who has been using our product for at least 1 month
Estimated time to complete: 10~12 mins

3. Conduct User Interview

To really understand why they use our product, we have to figure out further more about the user’s background, motivation, exact using process, and opinion on our product.

Why do they need our product?
What do they think about our product?

Main questions:
(1) Job title/ Job Description/Team size

To make sure interviewees’ responses are reliable, we have to check if their response corresponds to behavior data (the amount of usage, frequency) and survey responses.

Conclusion

With the persona we’ve created, we have better understand about user. After getting the research results, let's go back to the question: What can we do to improve the retention of our keyword research tool?

The 2 main different ways to use the keyword research tool from user research:

1

Users who need new content frequently.

Only use keyword research tool, doesn’t use keyword and landing page tracking tool. This kind of user needs new content very often, what they need is to find out as many related keywords as possible. They don’t think it necessary to keep tracking every keyword ranking on the landing page. E.g. Content Websites, web design companies, and new-built websites

2

Users who don’t update content frequently,
but focus on certain keywords’ ranking.

Use keyword and landing page tracking tools often, but they hardly use the keyword research tool. To this kind of user, their content doesn’t change often. They do focus on keyword research at first but don’t pay much effort to keyword research anymore after using it for a while. They only pay attention to the ranking of certain keywords and landing pages. E.g. Online Business Brochure/Catalog, some E-commerce Websites.

Proposal 1

We can see, that the type 2 user did use our keyword research function at first, but stopped using it after a while since their content doesn’t need so frequent changes. They leave our keyword tool and turn to the tracking tool. However, the user journey is not well connected. Even though there’re users who know how to turn to the tracking tool, some users actually don’t know there’s a keyword-landing-page ranking tracking function in our tool. It might be necessary to reinforce the combination between the research function and the ranking tracking function. We then propose to combine the research function and tracking function together to help improve user retention on our tool.

Proposal 2

Combining keyword research tools and keyword-landing-page tracking tools may be a good proposal, but it might take a lot of effort. Considering we don’t have many resources to do it, for now, we proposed another way to improve retention. After the survey and interview, we also found many users don’t really know how to use our tool correctly. Some gave up using certain functions because they don’t know how to use them, which might be one of the reasons they don’t come back and use them again. Therefore, we think it important to add an onboarding process to teach users how to use our tool. We proposed onboarding on product interface or onboarding email design might be feasible ways to help improve user retention.
